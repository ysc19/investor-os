# Day 4 — Dynamic Data & Risk Analysis

## Today's Goal

Make the stock data pipeline more flexible by allowing users to select different historical time periods, improve user input validation, and introduce the first measure of investment risk.

---

## What I Built

- Added dynamic time period selection (1d, 5d, 1mo, 3mo, 6mo, 1y, 2y, 5y, 10y, ytd, max)
- Added validation to prevent invalid user input
- Refactored stock data retrieval to accept a configurable period
- Renamed the price plotting function to better reflect its purpose
- Added volatility calculation using the standard deviation of daily returns
- Improved volatility output formatting as a percentage
- Continued improving the modular architecture of the project

---

## What I Learned

- Hardcoding reduces flexibility; parameterized functions make software reusable.
- User input should always be validated before processing.
- Volatility measures the variability of returns over time, not the difference between a day's high and low prices.
- Standard deviation is one of the most important measures of financial risk.
- Professional investors rely on engineered features rather than raw market prices.

---

## Biggest Insight

Today I realized that my project is slowly shifting from simply displaying stock prices to analyzing market behavior.

Instead of only asking "What is the current price?", Investor OS is beginning to answer questions like "How risky is this investment?" and "How has the stock behaved over time?"

That feels like an important milestone.

# 2026-07-06 — Day 5

## Objective

Continue improving the Stock Data Pipeline by making it more flexible, introducing another technical indicator, and writing cleaner, more maintainable code.

# What I Built

### Dynamic Time Period Selection

Supported periods:

- 1d
- 5d
- 1mo
- 3mo
- 6mo
- 1y
- 2y
- 5y
- 10y
- ytd
- max

Added input validation to prevent invalid entries.

---

### Exponential Moving Average (EMA)

Implemented a 5-Day Exponential Moving Average using pandas.

Unlike the Simple Moving Average, the EMA gives greater weight to recent prices, making it respond faster to changing market conditions.

---

### Technical Analysis Chart

Combined multiple indicators into a single chart.

The chart now displays:

- Closing Price
- 5-Day Simple Moving Average (SMA)
- 5-Day Exponential Moving Average (EMA)

This provides a much better visual comparison of price behavior and trend changes.

---

### Volatility

Added volatility calculation using the standard deviation of daily returns.

Learned that volatility measures how widely returns fluctuate over time rather than simply measuring the difference between a day's high and low prices.

Higher volatility generally indicates greater uncertainty and risk.

---

# Programming Concepts Learned

- Exponential Weighted Moving Average (`ewm`)
- Standard deviation (`std`)
- Input validation
- List membership (`in`)
- String formatting with `join()`
- Cleaner function naming
- Refactoring existing code without changing behavior

---

# Debugging

Encountered and fixed several common Python errors:

- Unterminated string literal
- ImportError after renaming a function
- Parentheses mismatch while slicing DataFrames

These reinforced the importance of reading error messages carefully and understanding where Python expects specific syntax.

---

# Software Engineering Lessons

Today's biggest lesson was that good software is not only about adding features.

It is also about:

- Choosing meaningful function names
- Keeping `main.py` simple
- Organizing related functionality together
- Improving readability without over-engineering

I also learned that although creating additional files is tempting, it is better to wait until the project becomes large enough to justify further modularization.

---

# Finance Lessons

I now understand the difference between:

- Daily Returns
- Simple Moving Average (SMA)
- Exponential Moving Average (EMA)
- Volatility

These indicators don't predict the future on their own—they help describe trend, momentum, and risk so that better investment decisions can be made.
